# Модуль: Реновация

## Назначение

Преобразовать намерение, сформированное аудитом, в завершённый проект реновации. Провести жителей через весь процесс: от расчёта экономии и субсидии Altum до юридически значимого голосования собственников и выбора подрядчика.

Этот модуль — основной источник выручки ALTEKO (комиссия 1–2% с каждой сделки).

---

## Точка входа

Пользователь в модуле аудита видит, что расходы на отопление его дома значительно превышают медиану. Модуль показывает предложение:

> «Дома, похожие на ваш, экономят в среднем €140 в месяц на отоплении после реновации. Хотите узнать, сколько это будет стоить вам?»

Это усреднённая цифра по сегменту — не персональный расчёт. Она доступна без email.

Нажатие на кнопку открывает модуль реновации. Если email уже введён при аудите — пользователь сразу видит персональный расчёт. Если нет — появляется форма:

> «Укажите email, чтобы получить расчёт именно для вашего дома: конкретную сумму экономии, долю субсидии Altum и срок окупаемости.»

## Подмодуль 1: ИИ-прогноз экономии

Рассчитывает ожидаемое изменение коммунальных расходов после энергоэффективной реновации.

**Входные данные:**
- Текущий класс энергоэффективности (из Kadastrs)
- Площадь и серия дома (из Kadastrs)
- Текущее потребление тепла (из модуля аудита)
- Климатическая зона / градусо-дни отопления (из погодного API)
- Целевой класс энергоэффективности после реновации (рассчитывается на основе стандартного улучшения для данной серии)

**Результаты:**
- Ежемесячные расходы на отопление до реновации (€)
- Ежемесячные расходы на отопление после реновации (€)
- Годовая экономия (€)
- Срок окупаемости (лет)
- Оценка роста стоимости недвижимости (%) — верифицировано: +10–11%, источник: Latvijas Banka DP 3/2025

**Модель:** Физическая тепловая модель в MVP на основе опубликованных норм потребления по классам энергоэффективности. В более поздних фазах — ML-модель на исторических данных проектов Altum.

## Подмодуль 2: Калькулятор субсидии Altum

Показывает жителю точно, что покрывает государственная субсидия и какова будет его доля в стоимости реновации.

**Входные данные:**
- Общая площадь дома
- Оценочная стоимость реновации на м² (по индексу строительных затрат CSP + нормы Altum)
- Текущие условия программы Altum (поддерживаются вручную, обновляются с altum.lv)

**Результаты:**
- Итоговая оценочная стоимость реновации (€)
- Доля, покрываемая Altum (%)
- Сумма гранта Altum (€)
- Доля жителя — итого (€)
- Доля жителя — на квартиру (€, по доле собственности)
- Ежемесячный платёж при финансировании на N лет

**Примечание:** Условия Altum меняются. Этот подмодуль требует активного сопровождения при обновлении программных условий.

## Подмодуль 3: Генератор документов

Генерирует все документы, необходимые для:
1. Проведения юридически значимого собрания собственников
2. Подачи заявки на субсидию Altum
3. Привлечения подрядчиков для составления смет

**Генерируемые документы:**
- Уведомление о собрании собственников (повестка, дата, место или дистанционный формат)
- Шаблон протокола собрания (соответствует требованиям Altum)
- Заявка на субсидию, предзаполненная данными дома из Kadastrs
- Техническое задание для тендера подрядчиков

> Это концепция «TurboTax для реновации» — жителям не нужно разбираться в бюрократических требованиях. Платформа автоматически генерирует корректные документы.

## Подмодуль 4: Электронное голосование

Собирает юридически значимое согласие собственников (≥50% по доле собственности), необходимое для заявки на субсидию Altum.

**Требования:**
- Юридически значимый результат — должен быть принят Altum, банками и судами
- Каждый голосующий должен быть верифицирован как текущий владелец квартиры
- Вес голоса = доля собственности (из Zemesgrāmata, а не просто число квартир)

**Методы аутентификации:**
- Smart-ID (предпочтительно — широко распространён, нативный мобильный)
- eParaksts mobile

**Логика голосования:**
1. Платформа загружает список собственников из Zemesgrāmata для данного дома
2. Каждый собственник квартиры получает ссылку для голосования (SMS / email)
3. Голосующий проходит аутентификацию через Smart-ID или eParaksts
4. Голос записывается с цифровой подписью
5. Платформа отслеживает: всего проголосовавших долей, за, против, не проголосовали
6. При достижении ≥50% общих долей «за»: порог пройден
7. Система генерирует подписанный протокол голосования

**Результат:** Юридически значимый протокол голосования с цифровыми подписями, готовый для подачи в Altum.

> **Правовой статус подтверждён:** Dzīvokļa īpašuma likums, ст. 20(7-8), поправки 2022 г. Altum принимает протоколы опроса (aptauja), включая электронные через BIS. Smart-ID и eParaksts квалифицируются как «drošs elektroniskais paraksts» по eIDAS. Единственная оговорка: собственники не должны были ранее постановить, что данный вопрос решается только на физическом собрании.

## Итоговый поток

```
Обнаружена аномалия аудита
    ↓
Пользователь нажимает «Рассчитать экономию от реновации»
    ↓
ИИ-прогноз экономии → показана экономия €/мес.
    ↓
Калькулятор Altum → показаны доля пользователя и грант Altum
    ↓
Пользователь нажимает «Начать реновацию»
    ↓
Генератор документов → создано уведомление о собрании
    ↓
Запущена кампания голосования → уведомлены собственники
    ↓
Собрано ≥50% согласия → сгенерирован протокол голосования
    ↓
Маркетплейс подрядчиков → начат тендерный процесс
    ↓
Выбран подрядчик → подписан контракт
    ↓
ALTEKO получает комиссию (1–2% от контракта)
```
