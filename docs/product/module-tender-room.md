# Модуль: Piegādātāju atlases telpa / Комната выбора поставщиков

> Старое название: «Маркетплейс подрядчиков». Код остаётся (`src/app/[locale]/contractors/`, `src/app/api/contractors/`, `src/app/api/tenders/`), но переименовано согласно `readiness-glossary.md`. Хардкод `1.5%` в `api/tenders/bid` и `api/tenders/select` подлежит удалению в отдельной итерации.

## Назначение

Связать дом с подрядчиками **прозрачно**, **без скрытых комиссий** и **без конфликта интересов**. ALTEKO не продаёт «победу подрядчика» и не получает success fee. ALTEKO предоставляет инструмент для дома и пространство для подрядчиков, но решение принимает дом.

**Главный принцип:** ALTUM требует документировать независимость выбора поставщиков, отсутствие конфликта интересов, опросить ≥5 потенциальных поставщиков, получить ≥2 независимых предложения. Tender Room автоматизирует именно этот стандарт.

---

## Какую проблему решает

После того как дом готов к подаче, ему нужно выбрать поставщиков. Сейчас это происходит так:

1. Управляющий (или активная biedrība) ищет подрядчиков по личным связям
2. Запросы отправляются 3–5 компаниям, чаще «знакомым»
3. Сметы приходят на email/мессенджер
4. Сравнение делается в Excel или на бумаге
5. Выбор обосновывается протоколом задним числом

**Проблемы:**
- Конфликт интересов: «свой» подрядчик
- Нет независимости предложений: связанные компании подают «конкурирующие» сметы
- Документация выбора слабая, ALTUM может вернуть на доработку
- Цена завышена из-за тонкой конкуренции
- Дом не может проверить связи компаний

ALTEKO даёт **прозрачную процедурную линию** под требования ALTUM.

## Каталог подрядчиков

### Профиль подрядчика

Публичный (виден всем):
- Юридическое название и регистрационный номер
- Контактные данные
- Специализации (утепление фасада, кровля, окна, отопление, лифты, проектирование, надзор)
- Географический охват
- Прошлые проекты (количество, типы)
- Verified-маркеры: Lursoft (действующая регистрация), BVKB (лицензии), история с ALTUM

В подписке Plus (€200/мес.):
- Расширенное портфолио с фото и кейсами
- Аналитика по тендерам
- Уведомления о новых тендерах в географии

### Верификация

Перед появлением в каталоге:
1. **Lursoft:** действующая регистрация, отсутствие активного процесса неплатёжеспособности
2. **BVKB:** наличие соответствующих строительных лицензий
3. **ALTUM-история:** упоминания в реестре завершённых проектов (если есть)
4. **Anti-fraud:** проверка по чёрным спискам IUB

Это автоматическая проверка, не «личное знакомство».

### Подписка подрядчика

| Тариф | Цена | Возможности |
|-------|------|-------------|
| **Basic** | €50/мес. | Публичный профиль, базовая верификация, уведомления о тендерах |
| **Plus** | €200/мес. | Расширенный профиль, фото и кейсы, приоритет в search results (не в выборе!), аналитика |

**Что подписка не даёт:**
- ❌ Гарантию участия в конкретном тендере
- ❌ Преимущество при выборе домом
- ❌ Скрытые скидки или success fee

**Это критичное отличие от v1.** В v1 «1.5% от победителя» = конфликт интересов. В v2 — фиксированная плата за видимость.

## Tender Process

### Шаг 1: Создание тендера

Дом или специалист создаёт тендер. Поля:
- Объём работ (категории, объёмы, технические требования)
- Бюджет (диапазон или максимум)
- Сроки выполнения
- Требования к опыту (минимум проектов аналогичного типа)
- Документы для участия (свидетельства, лицензии, страховки)
- Период подачи предложений (минимум 14 дней по требованиям ALTUM)

### Шаг 2: Приглашение поставщиков

ALTEKO **автоматически предлагает список ≥5 верифицированных** подрядчиков из географии и специализации, согласно требованиям ALTUM.

Дом может:
- Согласиться с автоматическим списком
- Добавить дополнительных подрядчиков (вручную)
- Удалить из списка (с обоснованием — фиксируется в audit trail)

Минимум 5 приглашений отправляется автоматически (требование ALTUM).

### Шаг 3: Подача предложений

Подрядчики получают уведомления (email + push в кабинете). Подают предложения через стандартизированную форму:
- Цена (детализация по статьям)
- Сроки выполнения
- Условия оплаты
- Гарантии
- Привлечение субподрядчиков (с указанием)
- Подтверждения отсутствия конфликта интересов (галочка + ALTEKO проверяет автоматически через Lursoft + IUB graph)

### Шаг 4: Сравнение предложений

UI показывает сравнительную таблицу:
- Цена с разбивкой
- Сроки
- Опыт (количество завершённых проектов того же типа)
- Risk indicators: связи с другими участниками тендера, связи с управляющим, частые победы у одного клиента

**Risk indicators не блокируют выбор**, они **информируют** дом и **сохраняются в protocol**.

### Шаг 5: Выбор и протокол

Дом (или уполномоченное лицо) принимает решение через Decision Campaign типа `SUPPLIER_SELECTION_DECISION`. Голосование собственников или решение правления с делегированными полномочиями.

ALTEKO генерирует **протокол выбора** соответствующий требованиям ALTUM:
- Список приглашённых
- Список подавших
- Сравнительная таблица
- Risk indicators
- Обоснование выбора
- Подписи

### Шаг 6: Контракт

Переговоры по контракту вне платформы (в Phase 1) или на платформе (в Phase 4).

## Supplier Risk Check

Это **не отдельный модуль**, а функция в Tender Process и в подписке Speciālista portfelis.

### Что проверяется (граф связей)

| Признак | Источник |
|---------|---------|
| Одинаковые адреса компаний | Lursoft |
| Одинаковые члены правления | Lursoft |
| Одинаковые patiesā labuma guvēji | Lursoft |
| Связь подрядчика с управляющим (apsaimniekotājs) | Lursoft + ALTEKO внутренние данные |
| Связь подрядчика с проектировщиком (если разные тендеры) | Lursoft + ALTEKO |
| Частые победы одного подрядчика у одного управляющего | ALTEKO внутренние данные |
| Подозрительно малое число предложений | ALTEKO статистика тендеров |
| Резкие скачки цены после контракта | (в Phase 3+) ALTEKO post-contract tracking |
| Жалобы в IUB | IUB open data |
| Дисквалификации или судебные споры | IUB + Lursoft |

### Как используется

- **Дом** видит индикаторы при сравнении предложений
- **Уполномоченное лицо** прикладывает report к протоколу выбора
- **ALTEKO** не выносит вердикта «этот подрядчик плохой» — даёт информацию, дом решает

Это и есть «**модуль прозрачности**» из концепции рва (см. concept.md).

## Что меняется относительно v1

| v1 | v2 |
|----|-----|
| «Маркетплейс подрядчиков» | **Piegādātāju atlases telpa** / **Комната выбора поставщиков** |
| Подрядчик платит 1.5% при победе | Подрядчик платит €50–200/мес. за профиль, без success fee |
| «Подрядчик берёт горячий лид» | «Подрядчик участвует в прозрачном процессе выбора, инициированном домом» |
| ALTEKO выбирает дом подрядчику | Дом выбирает подрядчика, ALTEKO даёт инструмент |
| Автоматическое уведомление верифицированных | Автоматическое предложение ≥5 верифицированных по требованиям ALTUM |
| Отсутствие проверки связей | Supplier Risk Check как часть процесса |
| Контракт вне платформы | Контракт вне платформы (Phase 1–3), на платформе (Phase 4) |

## Защита от манипуляций (без изменений из v1)

Подрядчикам запрещено:
- Напрямую связываться с собственниками для обхода тендера
- Предлагать вознаграждение членам правления
- Подавать заниженные сметы для победы с последующим пересмотром

Нарушения влекут удаление из каталога и публикацию в реестре нарушителей.

## Зависимости

- **Данные:** Lursoft, IUB open data, BVKB реестр лицензий, ALTEKO внутренние данные
- **Питает:** Building Readiness Score (компоненты `supplier_selection_status` и `procurement_transparency_score`)
- **Связан с:** Decision Campaigns (`SUPPLIER_SELECTION_DECISION`)
