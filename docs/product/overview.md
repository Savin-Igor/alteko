# Обзор продукта

## Что делает ALTEKO

ALTEKO — **Mājas gatavības platforma** / **Платформа готовности дома**. Помогаем многоквартирному дому в Латвии стать готовым к финансированию и ремонту.

Не заменяет государственные системы (BIS, mans.altum.lv, KPVIS), а **готовит дом до того, как он зайдёт туда**.

---

## Четыре режима продукта

Продукт предъявляет разный интерфейс четырём типам пользователей. Все режимы работают над одной и той же сущностью — **карточкой дома** — но показывают её через разные роли.

| Режим | Кому | Главная боль | Главная ценность |
|-------|------|--------------|-----------------|
| **Publiskais režīms** / Публичный режим | Любой посетитель | «Что вообще с моим домом?» | Карточка дома, публичная оценка, CTA → Readiness Report |
| **Iedzīvotāja režīms** / Режим жильца | Владелец квартиры | «Почему такой счёт?» | Бенчмаркинг счёта, аномалии, **trust artifact** для собрания |
| **Valdes režīms** / Режим правления | biedrība, mājas vecākais | «Как не упустить следующее окно финансирования?» | Дашборд дома, кампании решений, экспорт в BIS |
| **Speciālista režīms** / Режим специалиста | apsaimniekotājs, ESCO, projektu vadītājs | «Я веду 30 домов в Excel — это ад» | Портфель, pipeline, шаблоны, multi-user |

## User journey по режимам

### Publiskais režīms — путь публичного посетителя

**Шаг 1 — поиск адреса:**
Пользователь вводит адрес (Jāņa sēta autocomplete) или приходит по ссылке `/b/<token>`. ALTEKO показывает карточку дома с публичными данными:
- Серия, год постройки, площадь (VZD Building.ZIP)
- Энергокласс (BVKB BIS)
- Серия здания (по эвристике из `series-tiebreaker`)
- Базовый readiness score (на основе только публичных данных)

**Шаг 2 — финансовые сценарии:**
Под карточкой — таблица с 5 сценариями. Для каждого: статус окна (`Slēgts / Gaidāms / Atvērts`), оценочная стоимость, оценочная экономия (в режиме «диапазон»).

**Шаг 3 — CTA:**
Под сценариями: «Получить отчёт о готовности дома» — переход к платному Readiness Report (форма + email).

> **Без email:** видно карточку дома и финансовые сценарии в общем виде.
> **После email:** доступ к платному отчёту + персональные расчёты.

---

### Iedzīvotāja režīms — путь жильца

**Шаг 1 — приходит за «почему такой счёт»:**
Загружает PDF-счёт от управляющего. ALTEKO парсит через GPT-4o vision, показывает превью одной цифры:

> «Дом серии 119 в Пурвциемсе платит за отопление на 40% больше медианы похожих домов в районе. Достоверность данных: средняя.»

**Шаг 2 — email-гейт:**
Под цифрой:
> «Получите полный отчёт по всем категориям и сравнение с похожими домами — введите email.»

**Шаг 3 — полный отчёт как trust artifact:**
После email — отчёт. Структура:
1. Таблица расходов по категориям с пометкой «выше / ниже / в норме»
2. Сравнение с P25/P50/P75 сегмента (если ≥10 домов в сегменте)
3. Аномалии (`ABOVE_P75`, `TREND_INCREASE`, `SPIKE`)
4. **«Какие вопросы задать управляющему»** — список с обоснованиями
5. **«Какой следующий шаг для дома»** — ссылка на Readiness Score дома

**Шаг 4 — путь к собранию:**
Жилец скачивает PDF-отчёт «для собрания собственников» и приносит на встречу. Это и есть **trust artifact**.

> Жилец **не покупает** реновацию через ALTEKO. Он создаёт давление снизу, передаёт отчёт правлению — правление становится клиентом Valdes režīms.

---

### Valdes režīms — путь правления / biedrība

**Шаг 0 — заказ Gatavības atskaite (€300–900):**
Член правления заказывает разовый отчёт. Получает PDF + доступ к дому в платформе на 30 дней.

**Шаг 1 — дашборд дома:**
- Building Readiness Score (8 компонентов)
- Список незакрытых пунктов (`next_best_action`)
- История счетов (если жильцы загружали или CSV от управляющего)
- Финансовые сценарии с актуальным статусом окна

**Шаг 2 — Lēmumu kampaņa (Decision Campaign):**
Правление создаёт первое решение собственников: «решение о начале подготовки дома».
- Подготовка вопроса с объяснением простым языком (LV/RU)
- Загрузка списка собственников (CSV вручную, т.к. Zemesgrāmata API нет)
- Кампания уведомлений: email/SMS, статусы «за / против / не ответил»
- Сбор предварительных намерений (перед формальным голосованием)
- При готовности — экспорт в BIS Mājas lieta для официального опроса/собрания

**Шаг 3 — пакет документов:**
Платформа генерирует:
- Повестку собрания
- Шаблон протокола
- Доверенности (для уполномоченного лица)
- Согласия на обработку данных (GDPR)

**Шаг 4 — переход в Speciālista režīms:**
Если правление хочет полное сопровождение — заказывает Projekta sagatavošanas pakete у специалиста через платформу. Или подписка переходит в годовой формат Valdes darba telpa €30–100/мес.

---

### Speciālista režīms — путь специалиста

**Шаг 0 — подписка Speciālista portfelis (€100/мес. + per-house):**
apsaimniekotājs или ESCO регистрируется, добавляет свои дома (импорт CSV).

**Шаг 1 — портфель:**
Дашборд со всеми домами в портфеле:
- Сортировка по readiness score
- Цветовая индикация дедлайнов
- Pipeline проектов: где какой дом

**Шаг 2 — рабочее место по дому:**
Для каждого дома — полный набор инструментов из Valdes режима + специальные:
- Шаблоны документов с подстановкой данных
- Tender Room: запросы предложений, сбор смет, протокол выбора
- Lursoft/IUB-проверка поставщиков (Supplier Risk Check)
- Экспорт в mans.altum.lv (когда возможно)

**Шаг 3 — multi-user:**
Команда специалиста добавляет коллег с разными ролями (читать / редактировать / только свои дома).

**Шаг 4 — Projekta sagatavošanas pakete:**
Когда дом готов к подаче — специалист продаёт дому пакет за €2 000–8 000 (или это цена-подписка специалиста, в зависимости от структуры контракта).

## Язык

Продукт двуязычный: **латышский (default, без префикса URL) и русский (`/ru/...`)**.

В UI **никогда не используются английские термины**. Все названия переведены через `docs/reference/readiness-glossary.md`.

В советских домах высокая доля русскоязычных жителей. Достичь 50%+ согласия собственников для голосования невозможно без RU-локализации.

## Платформа

- Веб (десктоп + мобильный браузер) — Phase 1–4
- Мобильное приложение — после €1M ARR (не в дорожной карте текущей фазы)
