# Дизайн-система ALTEKO

Справочник по токенам, компонентам и правилам реализации UI.

Принципы UX и текстовые вайрфреймы — в `product/design.md`. Этот документ описывает техническую реализацию.

---

## 1. Стек

| Решение | Выбор | Обоснование |
|---------|-------|-------------|
| CSS-фреймворк | Tailwind CSS v3 | Уже в проекте |
| Компонентная библиотека | Не используется (shadcn/ui отклонён) | 12 компонентов не оправдывают 40+ Radix-файлов и ~50KB bundle |
| Шрифт | Системный стек | `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif` |
| Кастомный шрифт | Нет | Системные шрифты лучше читаются для аудитории 55+, не добавляют latency |
| Тёмная тема | Нет | Белый фон воспринимается целевой аудиторией как официальный и надёжный |

## 2. Цветовые токены

Определяются в `tailwind.config.ts`.

### Семантические цвета

| Токен | DEFAULT | light | dark | Применение |
|-------|---------|-------|------|------------|
| `primary` | `#2563EB` | `#EFF6FF` | `#1D4ED8` | Основные CTA, активные состояния |
| `danger` | `#DC2626` | `#FEF2F2` | — | Ошибки, критические отклонения (>20%) |
| `warning` | `#EA580C` | `#FFF7ED` | — | Предупреждения, умеренные отклонения (5–20%) |
| `success` | `#16A34A` | `#F0FDF4` | — | Успех, готовность, открытое окно финансирования |

### Нейтральные цвета

Стандартная шкала Tailwind: `gray-50` → `gray-900`. Нестандартные серые не вводятся.

### Токены для статуса окна финансирования

| Токен | Цвет | Применение |
|-------|------|-----------|
| `funding-open` | `success` | Окно открыто (ALTUM remonta aizdevums, банк) |
| `funding-expected` | `warning` | Окно ожидается (SCF 2026-2032) |
| `funding-closed` | `gray-500` | Окно закрыто (ALTUM 2021-2027) |
| `funding-unknown` | `gray-400` | Статус неизвестен |

### Токены для достоверности данных (`dataConfidence`)

| Токен | Иконка | Цвет | Применение |
|-------|--------|------|-----------|
| `dc-public` | 🌐 | `gray-500` | PUBLIC_DATA — только публичные данные |
| `dc-user` | ✏️ | `primary` | USER_UPLOADED — загружено пользователем |
| `dc-board` | ✓ | `success` | BOARD_VERIFIED — подтверждено правлением |
| `dc-pro` | ★ | `#D97706` (gold) | PROFESSIONAL_VERIFIED — подтверждено специалистом |

### Токены для Readiness Score (0–100)

| Диапазон | Цвет |
|----------|------|
| 0–25 | `danger` |
| 26–50 | `warning` |
| 51–75 | `primary` |
| 76–100 | `success` |

## 3. Типографика

| Параметр | Значение |
|----------|---------|
| Базовый размер | 16px |
| Минимум для интерактивных элементов | 12px |

### Классы метрик данных

Применяются для отображения ключевых числовых показателей (например, «+43% к норме»).

| Класс | Размер | Font-weight | Применение |
|-------|--------|-------------|------------|
| `text-metric-xl` | 2.5rem | 700 | Основное число страницы |
| `text-metric-lg` | 1.875rem | 700 | Вторичные метрики |
| `text-metric` | 1.5rem | 700 | Числа в карточках |

Основное число на странице (например, «+43% к норме») — `text-metric-xl` в семантическом цвете.

## 4. Ширина контента и отступы

### Стандарты max-width

| Элемент | Класс | px | Применение |
|---------|-------|----|-----------|
| Хедер/Навигация | `max-w-5xl` | 1024 | `SiteHeader`, единственный |
| Футер | `max-w-4xl` | 896 | `SiteFooter`, единственный |
| Контентные секции | `max-w-3xl` | 768 | Основной контент страниц |
| Hero-блок | `max-w-xl` | 576 | Центрированный CTA-блок в hero |
| Узкие карточки/формы | `max-w-lg` | 512 | Одиночные формы, sample-карточки |
| CTA-блоки | `max-w-lg` | 512 | Призывы к действию в нижней части страниц |

**Правило:** не смешивать произвольные ширины внутри одной страницы. Секции одной страницы используют один стандарт (`max-w-3xl`), за исключением намеренно узких блоков (hero, CTA).

### Горизонтальные отступы

Все секции: `px-4` — обеспечивает 16px margin на мобильных без разрыва макета.

### Вертикальные отступы

| Тип секции | Класс |
|-----------|-------|
| Основные секции | `py-14` |
| Компактные секции (цифры, источники) | `py-12` |
| Hero и финальный CTA | `py-16` |

## 5. Компоненты

Расположены в `src/components/ui/`. Экспортируются через `src/components/ui/index.ts`.

**Правило:** каждый компонент ≤100 строк, без сторонних зависимостей.

### Оболочка сайта (Shell)

Два компонента формируют единую оболочку публичных страниц.

#### `SiteHeader`

Файл: `src/components/ui/SiteHeader.tsx`

- Sticky, `z-50`, белый фон с нижней границей
- Max-width контейнера: `max-w-5xl`
- Навигация (после rewrite): **Mājas gatavība** / **Готовность** (`/`) · **Finansējums** / **Финансирование** (`/financing`) · **Speciālistiem** / **Специалистам** (`/dashboard/portfolio` или маркетинговая `/specialists`) · **Blogs** / **Блог** (`/blog`)
- CTA-кнопка: «Atrast māju →» / «Найти дом →» → `/#hero`
- Переключатель языка: `LangToggle` (LV / RU)
- Мобильное меню: hamburger, раскрывается вниз
- Активное состояние: `text-primary bg-primary-light` на текущем маршруте

**Никаких английских пунктов меню** — все только LV/RU. См. `docs/reference/readiness-glossary.md`.

#### `SiteFooter`

Файл: `src/components/ui/SiteFooter.tsx`

- Max-width контейнера: `max-w-4xl`
- Сетка: `grid-cols-2 md:grid-cols-4`, 4 колонки на десктопе

| Колонка | Содержимое |
|---------|-----------|
| Бренд | Лого ALTEKO + Mājas gatavības platforma / Платформа готовности дома |
| Сервисы | Mājas gatavība, Finansējuma scenāriji, Izdevumu izpratne, Lēmumu kampaņas, Blogs |
| Speciālistiem | Speciālista portfelis, Piegādātājiem (`/contractors`), Reģistrēties, Kā tas strādā |
| Правовое | Политика конфиденциальности, Условия использования, info@alteko.lv |

- Нижняя строка: copyright `© 2025 ALTEKO · Latvija`
- Все ссылки — locale-aware (`Link` из `@/i18n/navigation`)
- i18n: namespace `footer` в `ru.json` / `lv.json`

#### Шаблоны страниц

Страницы делятся на два типа по наличию оболочки:

**Тип A — Публичные маркетинговые страницы** (`SiteHeader` + `SiteFooter`):

| Маршрут | Страница |
|---------|---------|
| `/` | Главная (Mājas gatavības platforma) |
| `/financing` | Финансовые сценарии (новая) |
| `/specialists` | Лендинг для специалистов (новая, Phase 2) |
| `/blog` | Список статей |
| `/blog/[slug]` | Статья |
| `/contractors` | Лендинг для подрядчиков (Tender Room) |
| `/contractors/register` | Регистрация подрядчика |
| `/privacy` | Политика конфиденциальности |
| `/terms` | Условия использования |

**Тип B — Функциональные страницы** (только `PageHeader`, без `SiteFooter`):

| Маршрут | Заголовок | Причина |
|---------|-----------|---------|
| `/audit/upload` | breadcrumb | Мастер-сценарий загрузки счёта |
| `/audit/preview` | breadcrumb | Ожидание анализа |
| `/audit/report/[id]` | breadcrumb | Trust artifact: вопросы управляющему + ссылка на готовность |
| `/financing` | breadcrumb | Финансовые сценарии (5 вариантов) |
| `/readiness/order` | breadcrumb | Заказ Gatavības atskaite (платный отчёт) |
| `/dashboard/*` | admin | Valdes darba telpa и Speciālista portfelis |
| `/voting/[campaignId]` | breadcrumb | Decision Campaign |
| `/auth/*` | minimal | Минимальный контекст |
| `/building/[code]` | breadcrumb | Карточка дома + readiness score |
| `/b/[token]` | minimal | Реферальная карточка дома |

#### Компоненты разметки

| Компонент | Назначение | Props |
|-----------|-----------|-------|
| `PageHeader` | Шапка функциональной страницы | `variant`: `minimal` / `breadcrumb` / `admin`; `backHref?`; `backLabel?` |
| `LangToggle` | Переключатель LV/RU | — (берёт текущий locale из роутера) |
| `InfoBanner` | Информационная плашка | `variant`: `info` / `warning` / `success` / `error` |
| `EmptyState` | Состояние «нет данных» | `title`; `description`; `action?` (`{label, href}`) |

Варианты `PageHeader`:
- `minimal` — только лого
- `breadcrumb` — кнопка «назад» + лого
- `admin` — лого + панель правления

`LangToggle` — минимальная высота 48px (требование доступности).

`InfoBanner` заменяет разрозненные `bg-orange-50 div`-ы по всему приложению.

### Компоненты данных

| Компонент | Назначение | Props |
|-----------|-----------|-------|
| `StatCard` | Карточка с большим числом | `label`; `value`; `unit?`; `variant`: `neutral` / `danger` / `warning` / `success`; `size`: `sm` / `md` / `lg` |
| `CategoryRow` | Строка таблицы расходов | `label`; `valuePerM2`; `deviationPct`; `hasData` |
| `Badge` | Статусный ярлык | `status`: `active` / `done` / `cancelled` / `pending` |
| `ProgressBar` | Прогресс-бар | `value` (0–1); `variant`; `label?` |

`StatCard` использует классы `text-metric-*` в зависимости от `size`.

`CategoryRow` показывает цветную точку (`.status-dot-*`) + значение + отклонение.

### Компоненты взаимодействия

| Компонент | Назначение | Props |
|-----------|-----------|-------|
| `FileDropzone` | Зона загрузки файла | `onFile`; `file?`; `accept?`; `maxSizeMB?`; `onError?` |
| `StepProgress` | Прогресс многошагового процесса | `steps: Array<{key, label, status}>` |

`FileDropzone` использует `<div role="button">` вместо `<button>` — исправляет drag-and-drop на iOS Safari.

`StepProgress` статусы: `done` / `active` / `pending` / `error`. Иконки — SVG, не emoji.

| Класс | Применение |
|-------|-----------|

## 7. Интерактивные состояния

Каждый интерактивный элемент должен иметь все 5 состояний.

| Состояние | Реализация |
|-----------|-----------|
| Default | Базовые стили |
| Hover | `hover:bg-primary-dark` (кнопки), `hover:bg-gray-50` (ссылки/ghost) |
| Focus | `focus:ring-2 focus:ring-primary` (2px синее кольцо) |
| Active/Pressed | `active:opacity-90` |
| Disabled | `opacity-50 cursor-not-allowed` |
| Loading (кнопки) | Inline spinner: `w-4 h-4 border-2 border-white border-t-transparent rounded-full animate-spin` |

## 8. Иконографика

| Правило | Детали |
|---------|--------|
| Иконки | SVG inline, не emoji |
| Причина | Emoji рендерятся непоследовательно на Android/iOS |
| Исключение | Emoji разрешены только в тексте WhatsApp/Telegram сообщений для шаринга |
| Статусные индикаторы | CSS классы `.status-dot-*`, не emoji |

## 9. Запрещённые паттерны

Следующее не используется в ALTEKO намеренно:

| Запрет | Обоснование |
|--------|------------|
| Градиентные фоны | Снижают читаемость, выглядят неофициально |
| Фотографии в hero-секциях | Не соответствуют тону официального сервиса |
| Floating action buttons | Плохо работают на мобильных у аудитории 55+ |
| Bottom sheet / drawer | Нетипичны для целевой аудитории |
| Анимационные библиотеки (Framer Motion и др.) | Избыточный bundle, не добавляют ценности |
| Тёмная тема | Хуже воспринимается аудиторией 55+; белый фон = официальность |
| Горизонтальные scroll-карусели | Неинтуитивны для аудитории 55+, плохо работают на тач |

## 10. Доступность (a11y)

Целевая аудитория 55+ требует повышенного внимания к доступности.

| Требование | Реализация |
|-----------|-----------|
| Минимальный touch target | `min-h-[48px]` на всех кнопках и полях ввода |
| Контраст текста | Основной текст `gray-900` на белом — ratio >7:1 |
| Focus indicator | `focus:ring-2 focus:ring-primary` (не убирать `outline: none` без замены) |
| Кнопки без иконок | Всегда с текстовым `aria-label` |
| Spinner | `aria-hidden="true"` + текстовая метка рядом |
| Язык | `<html lang="{locale}">` — задаётся в `[locale]/layout.tsx` |
