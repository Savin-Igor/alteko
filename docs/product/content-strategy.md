# Контент-стратегия ALTEKO

Этот документ — единая точка отсчёта для всего контента платформы. Перед тем как менять копирайт на любой странице, добавлять новый блок или планировать статью в блог, сначала сверяйтесь с этим документом.

---

## 1. Нарративный стержень

Пользователь проходит через три эмоциональных стадии. Каждый кусок контента принадлежит ровно одной из них.

| Стадия | Эмоция | Вопрос пользователя | Ответ ALTEKO |
|--------|--------|---------------------|--------------|
| **Злость** | Раздражение от высокого счёта | «Меня обманывают?» | «Да. Вот доказательство.» |
| **Надежда** | Ощущение возможности | «Это можно исправить? Хватит денег?» | «Да. Государство платит 49%. Вот ваши числа.» |
| **Действие** | Готовность к шагу | «Как это реально запустить?» | «Мы ведём весь процесс — голосование, документы, подрядчик.» |

**Главное правило:** контент о реновации не должен появляться до того, как стадия Злости закрыта. Пользователь, который ещё не видел свой аудит, не готов слышать про субсидии Altum — это информация без контекста.

## 2. Целевые аудитории

| ID | Аудитория | Боль | Точка входа | Стадия воронки |
|----|-----------|------|-------------|----------------|
| A1 | Собственник квартиры | «Почему такой высокий счёт?» | Главная, блог, поиск Google | Злость |
| A2 | Председатель правления biedrības | «Нужен инструмент для переговоров и реновации» | Главная (блок «Правлениям»), дашборд | Злость → Действие |
| A3 | Активный жилец-инициатор | «Хочу реновацию, не знаю как начать» | /renovation, SEO по субсидии Altum | Надежда |
| A4 | Сосед по шарлинку | «Сосед прислал ссылку — надо разобраться» | /b/[token], /building/[code] | Злость |
| C  | Подрядчик-строитель | «Нужны квалифицированные лиды» | /contractors | Отдельная воронка |

Сегменты A1–A4 идут по основной воронке. Сегмент C — полностью отдельная воронка (/contractors, /contractors/register).

## 4. Email-гейт: что открыто, что закрыто

**Открыто без email:**
- Поиск адреса и карточка дома
- Загрузка счёта и превью анализа
- Одно число-превью (% отклонения)
- Страница /renovation (введение)
- Голосование через Smart-ID (/voting/[id])
- Маркетинговые страницы

**Закрыто (требует email):**
- Полный отчёт с разбивкой по категориям
- Персональный расчёт реновации

**Требует авторизации (NextAuth):**
- Дашборд правления (/dashboard)

```

  ↓ [адресный поиск]
/building/[code]
  ↓ [Загрузить счёт]
/audit/upload
  ↓
/audit/preview
  ↓

  ↓ [CTA реновация]
/renovation/preview?building=[code]
  ↓
/renovation/calculate/[id]
  ↓ [CTA голосование]
/dashboard → /voting/[id]

Отдельный путь (реферал):
/b/[token] → /building/[code] → /audit/upload

Блог:
/blog/* → / или /building/[code] (в каждой статье — CTA проверить дом)

B2B:
/contractors → /contractors/register
```

## 9. Правила локализации

- Весь копирайт выходит одновременно на LV и RU
- LV — первичный язык для SEO (Google.lv)
- RU — равный по качеству, написан в естественном регистре, не дословный перевод
- Не использовать бюрократический тон: «жилищно-коммунальное хозяйство» → «счёт за отопление»
- Термины Altum, Kadastrs, Smart-ID, eParaksts, biedrība не переводятся — остаются в обоих языках
- Все цифры идентичны в обеих версиях — проверять по `docs/reference/key-facts.md`

## 10. Метрики

| Метрика | Ориентир | Где измерять |
|---------|----------|--------------|
| Email capture rate (отчёт аудита) | ≥40% пользователей, увидевших гейт | Аналитика событий |
| Аудит → переход на реновацию | ≥25% завершивших аудит | Аналитика событий |
| Органический трафик блога | Рост месяц к месяцу с 3-го месяца | Google Search Console |
| Главная → /audit/upload | ≥15% посетителей главной | Аналитика воронки |

Ориентиры, не жёсткие KPI. Измерять ежемесячно.
