# Дизайн ALTEKO

UX-документ: принципы, экраны, сценарии и состояния.

---

## Часть 1. Принципы дизайна

### Кто наш пользователь

Владелец квартиры в советском доме. Часто 55+. Смотрит с телефона. Не доверяет незнакомым сайтам. Привык к государственным порталам (latvija.gov.lv, e-veseliba.lv) — они выглядят официально, но неудобно. Мы должны выглядеть надёжнее них, но работать легче.

Второй пользователь — председатель правления biedrības. 45–65 лет, привык к Excel и PDF. Ценит конкретику: цифры, отчёты, документы. Не терпит «умных» интерфейсов.

---

### Принцип 1. Одно действие — один экран

Пользователь не должен думать «что здесь надо сделать». На каждом экране одна кнопка или одно поле. Всё остальное — контекст, который помогает принять решение.

**Нарушение:** форма с 5 полями сразу. Пользователь закроет страницу.
**Правильно:** «Введите адрес вашего дома» → кнопка → следующий экран.

---

### Принцип 2. Говори числами, не словами

«Вы переплачиваете» — абстракция. «Ваш дом переплачивает за отопление на 43% больше, чем дома той же серии в вашем районе» — факт.

Каждый экран должен содержать одну ключевую цифру. Крупно. Без лишних слов вокруг.

---

### Принцип 3. Доверие строится до email-гейта

Пользователь должен убедиться, что продукт работает, прежде чем дать email. Email-гейт — это не барьер, это обмен. Если до гейта не показать реальную ценность — конверсия будет нулевой.

**Правило:** превью (одно число) должно быть достаточно убедительным, чтобы захотеть узнать детали.

---

### Принцип 4. Мобиль — первичен

Большинство пользователей придут с телефона. Дизайн строится для экрана 390px, потом адаптируется для десктопа. Не наоборот.

**Следствия:**
- Кнопки минимум 48px высотой (palec, не курсор)
- Шрифт основного текста минимум 16px
- Никаких hover-эффектов как единственного сигнала
- Поля ввода — полная ширина

---

### Принцип 5. Текст на языке пользователя

Двуязычность (LV/RU) — не опция, это требование рынка. Переключение языка — в шапке, всегда видно. Все тексты интерфейса переведены полностью, без «рыбы».

**Дополнительно:** избегать юридического и технического языка. «Кадастровый номер» → «номер дома в реестре». «Бенчмаркинг» → «сравнение с похожими домами».

---

### Принцип 6. Ошибки объясняются, не обвиняют

Если что-то пошло не так — виновата система, не пользователь. Текст ошибки говорит что случилось и что делать дальше.

**Плохо:** «Ошибка 422. Файл недействителен.»
**Хорошо:** «Не удалось прочитать файл. Убедитесь, что это PDF-счёт от управляющей компании, и попробуйте ещё раз.»

---

### Принцип 7. Прогресс должен быть виден

Длинные операции (загрузка PDF, анализ GPT-4o — занимает 5–15 сек) требуют индикатора прогресса с описанием шага. Молчащий спиннер пугает пожилого пользователя — он думает, что сломалось.

**Шаги для показа во время анализа PDF:**
1. «Загружаем файл...»
2. «Читаем данные счёта...»
3. «Сравниваем с похожими домами...»
4. «Готово»

---

### Цветовая философия (без полного дизайн-системы)

| Сигнал | Цвет | Когда |
|--------|------|-------|
| Переплата / аномалия | Оранжевый / красный | Показываем проблему — без паники |
| Экономия / позитив | Зелёный | Потенциал реновации, успех голосования |
| Нейтральный | Серый | Данные без оценки |
| CTA (главное действие) | Синий | Одна кнопка на экране |

Фон — белый. Никаких тёмных тем — пожилые пользователи хуже читают светлое на тёмном.

### Экран 11. Расшаренная страница (что видит сосед)

Это страница, куда попадает сосед по ссылке из WhatsApp/Telegram. Показывает данные **дома** — не личный отчёт отправителя. Никакой личной информации не раскрывается.

URL: `/b/{building_token}` — короткий, без кадастрового номера в явном виде.

```
┌─────────────────────────────────────────┐
│  ALTEKO                    [LV] [RU]    │
├─────────────────────────────────────────┤
│                                         │
│  Ваш сосед проверил расходы            │
│  вашего дома                           │
│                                         │
│  Brīvības iela 55, Rīga               │
│  Серия 119 · 1978 г. · Класс E        │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │                                 │    │
│  │   Отопление: +43% к норме       │    │
│  │   Потенциальная переплата:      │    │
│  │   ~€840/год на дом              │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
│                                         │
│  Это общие данные по дому.             │
│  Загрузите ваш счёт — покажем          │
│  детали именно по вашему подъезду      │
│                                         │
│  [Загрузить свой счёт — бесплатно]     │
│                                         │
│  Бесплатно · Без регистрации · 30 сек  │
└─────────────────────────────────────────┘
```

**Ключевые решения:**

- **«Ваш сосед»** — анонимно. Имя отправителя не раскрывается никогда. Это принципиально: пожилые не будут делиться, если знают что «засветятся»
- **Данные дома, не квартиры** — то, что шарится, это бенчмарк по дому в целом (публичные агрегированные данные), не личный счёт
- **Адрес и число сразу** — сосед мгновенно узнаёт свой дом и видит конкретную цифру. Это и есть ценность, за которой он кликнул
- **«По вашему подъезду»** — небольшая локализация обещания. Не врём (реально показываем по дому), но создаём ожидание персональности
- **Один CTA** — загрузить счёт. Не «узнать подробнее», не «зарегистрироваться». Сразу к делу
- **Технически** — страница `/b/{token}` это карточка дома (Экран 3) + превью (Экран 6), уже построенные. Никакого нового компонента

**Что НЕ показывается:**
- Кто поделился
- Содержимое исходного PDF
- Разбивка по категориям (это за email-гейтом)
- Данные конкретной квартиры отправителя

### Механика шаринга

**Текст для WhatsApp (RU):**
> Проверил расходы нашего дома на ALTEKO. Переплачиваем за отопление на 43%. Посмотри: alteko.lv/b/abc123

**Текст для WhatsApp (LV):**
> Pārbaudīju mūsu mājas izdevumus ALTEKO. Pārmaksājam par apkuri 43%. Skaties: alteko.lv/b/abc123

**Правила для текста:**
- Процент подставляется из реального результата, не фиксированный
- Ссылка — короткая, без параметров в явном виде
- Telegram — тот же текст, Telegram хорошо рендерит превью ссылок
- «Скопировать» — просто URL в буфер, без текста (пользователь пишет своё)

**Почему WhatsApp первым:**
В Латвии WhatsApp — основной канал общения в домовых группах для всех возрастов. Домовые чаты в WhatsApp есть у большинства многоквартирных домов с активным правлением. Это естественный канал для «новостей по дому».

### Шаринг (вирусный поток)

| Сценарий | Экран | Поведение |
|---------|-------|-----------|
| Happy path | Экран 8 → WhatsApp | Открывается WhatsApp с предзаполненным текстом и ссылкой. Сосед кликает → Экран 11 → стандартный поток |
| Пользователь копирует ссылку | Экран 8 | Копируется `/b/{token}`. Всплывает тост: «Ссылка скопирована» на 2 сек |
| Сосед переходит по ссылке, дом тот же | Экран 11 | Видит адрес, узнаёт свой дом, кликает CTA → стандартный поток с Экрана 4 |

| Ссылка устарела / токен не найден | Экран 11 | «Эта ссылка устарела. Найдите свой дом:» + поле адреса прямо на странице |

| Дом с нормальными расходами (< +10%) | Экран 11 | «Ваш дом тратит в норме. Но детали — в полном отчёте.» Всё равно ведём к загрузке PDF |

### Общие состояния (все модули)

| Состояние | Поведение |
|-----------|-----------|
| Загрузка | Скелетон-экран (не пустой белый экран) |

| Успех | Подтверждение действия + следующий шаг (не тупик) |
