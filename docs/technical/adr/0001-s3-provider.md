## Контекст

ALTEKO хранит пользовательские файлы трёх категорий:

| Категория | Примеры | Чувствительность | Срок жизни |
|---|---|---|---|
| Документы дома | энергосертификат, технический паспорт, протокол собрания | **GDPR special category** (документы с подписями) | бессрочно |
| Загрузки счетов | PDF от управляющей компании | персональные данные | 90 дней (после анализа удаляются) |
| Сгенерированные отчёты | Readiness Report PDF | производный артефакт | 365 дней |
| Медиа CMS | hero-картинки, фото из блога | публичные | бессрочно |

Локальная разработка использует MinIO в Docker. Для production нужно решить — какой провайдер.

### Критерии решения (по приоритету)

1. **EU data residency** — GDPR Art. 28 требует размещения в ЕЭЗ или DPA. Документы собственников квартир — это особо чувствительные данные.
2. **EUR-биллинг** — без FX-сюрпризов и счетов в валюте, не совпадающей с регистрацией SIA.
3. **Durability ≥ 99.999999999% (11×9)** — стандарт object storage. Single-host MinIO не дотягивает.
4. **Стоимость на горизонте 24 месяцев** — прогноз: ~10 GB к концу года 1, ~100 GB к концу года 2 (PDF + изображения).
5. **Операционная простота** — меньше движущихся частей = ниже шанс инцидента.
6. **Совместимость с экосистемой** — VPS уже на Hetzner; одинаковый registrar упрощает биллинг и операционку.

## Рассмотренные варианты

### A. Hetzner Object Storage *(рекомендуется)*

**За:**
- Тот же провайдер, что VPS — единый счёт в EUR, единая поддержка.
- Регионы Falkenstein (DE) и Helsinki (FI) — обе в ЕЭЗ.
- S3-совместимый API (`s3-compatible`).
- Тариф ~€5.96/TB/месяц + €1/TB egress (бесплатно между Hetzner Cloud и Object Storage в одном регионе).
- DPA подписывается одним документом вместе с VPS.

**Против:**
- На май 2026 года статус GA в нужном регионе требует подтверждения перед provisioning. Если в выбранной локации beta — переоценить риск.
- Меньше готовых SDK-интеграций и инструментов, протестированных против него (по сравнению с AWS S3).

**Прогноз стоимости (год 2):** 100 GB × €0.00596 = **~€0.60/месяц**. Незаметно в общем счёте.

---

### B. MinIO self-hosted на VPS

**За:**
- Нулевая дополнительная стоимость.
- Полный контроль.

**Против:**
- VPS — 4 GB RAM, делится с MezaData, latviaforestbot, ALTEKO. +1 контейнер увеличит RAM-давление.
- **Single-disk durability** — потеря диска = потеря всего. Для документов собственников квартир с GDPR-подписями это неприемлемо.
- Бэкапы — наша забота. Нужен второй хост или внешний B2/S3 для off-site.
- Восстановление после инцидента — ручное.

**Вердикт:** не подходит как primary. Можно оставить как fallback для dev/staging.

---

### C. Backblaze B2

**За:**
- Самый дешёвый: ~$6/TB/месяц (≈€5.55).
- S3-совместимый API.

**Против:**
- USD-биллинг.
- EU-residency опциональна и зависит от выбора bucket region; не во всех регионах.
- Юридический контур не Hetzner — нужен отдельный DPA, отдельная юр-проверка.

**Вердикт:** дешевле на 8–10%, но добавляет юридическую и операционную сложность. Не оправдано на нашем объёме.

---

### D. AWS S3 (eu-central-1)

**За:**
- Золотой стандарт. Любой инструмент, любой SDK.
- 11 nines durability официально.
- Frankfurt — в ЕЭЗ.

**Против:**
- USD-биллинг.
- ~$23/TB/месяц + egress — дороже Hetzner в 4 раза.
- Усложняет SIA-бухгалтерию (USD invoice).

**Вердикт:** избыточно для текущей стадии. Может быть рассмотрено при переходе на CDN-heavy медиа (год 3+).

---

## Решение

**Выбран вариант A — Hetzner Object Storage** (регион Falkenstein как первый выбор, Helsinki как fallback при проблемах с Falkenstein).

Обоснование:
- Все 6 критериев удовлетворены.
- Самая низкая операционная сложность: один провайдер для VPS + хранилища, один счёт, один DPA.
- EUR-биллинг совпадает с регистрацией SIA Латвия.
- На прогнозе год 2 (100 GB) счёт ~€7/год — пренебрежимо мал.

**Условие активации:** перед созданием access key подтвердить, что выбранная локация имеет статус GA (не beta) на момент provisioning. Если beta — пересмотреть вариант D с переходом, когда станет GA.

## Раскладка bucket'ов

| Bucket | Назначение | Lifecycle | Public |
|---|---|---|---|
| `alteko-documents` | документы домов (энергосертификаты, технические паспорта, OWNER_LIST, OWNER_DECISIONS, GDPR_CONSENTS) | бессрочно, soft-delete на 30 дней | приватный, presigned URL |
| `alteko-uploads` | загрузки счетов от пользователей (PDF) | удаление через 90 дней если не привязан к ExpenseReport | приватный |
| `alteko-reports` | сгенерированные Readiness Report PDF | удаление через 365 дней | приватный, presigned URL |
| `alteko-media` | hero-изображения, медиа Payload CMS | бессрочно | публичный (read-only) |

Все bucket'ы — EU region (Falkenstein), серверная шифрация AES-256, versioning включён для `alteko-documents` (защита от случайного перезаписания подписанных PDF).

## Последствия

### Позитивные

1. Декомпозиция bucket'ов по lifecycle упрощает GDPR Art. 17 (право на удаление) — `alteko-uploads` сам очищается.
2. Единая Hetzner-инфра уменьшает операционную нагрузку.
3. EUR-биллинг прозрачен для бухгалтерии SIA.

### Негативные

1. Привязка к Hetzner: при миграции в другой ЦОД потребуется перенос данных (но S3 API совместим — миграция стандартна).
2. У Hetzner на момент решения нет встроенной replication между регионами — для disaster recovery нужно реализовать самим (cron-скрипт `rclone sync` в Helsinki, сценарий для второй фазы).

### Меры

1. Cron-скрипт `rclone sync` из Falkenstein в Helsinki — раз в сутки. Реализовать вместе с #139 как отдельный issue.
2. В Prisma добавить миграцию: на каждый файл записать `s3Key` и `s3Bucket` — на случай мульти-bucket'ной маршрутизации.
3. Перед первым production-деплоем (`v0.1.0`) проверить через `aws s3 ls` (или `mc ls`) что все 4 bucket'а доступны и lifecycle применён.

## Альтернативы для будущего

- **Год 3+, CDN-heavy медиа** → рассмотреть AWS S3 + CloudFront, или Bunny Storage.
- **Если beta-статус в Falkenstein затянется** → AWS S3 eu-central-1 как временный мост (D).
- **Если издержки вырастут на 2 порядка** → переоценить B2 (C).
