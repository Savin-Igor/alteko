# Дорожная карта

## Фазы

### Фаза 1 — Сбор данных (месяцы 1–4)

**Цель:** Запустить AI-аудит (GPT-4o vision), собрать данные по 500+ домам, сформировать начальные бенчмарки.

**Что выходит:**
- Загрузка и парсинг PDF (GPT-4o vision — один вызов, без отдельного OCR)
- Базовый отчёт о расходах (структурированная таблица по категориям)
- Бесплатный тариф для всех пользователей
- Внутренняя база бенчмаркинга начинает наполняться

**Что НЕ выходит:**
- Сравнение между домами (требует минимального объёма данных)
- Модуль реновации
- Электронное голосование
- Маркетплейс подрядчиков

**Метрика успеха:** 500+ домов с хотя бы одним загруженным отчётом о расходах.

**Ключевая активность:** Партнёрство с жилищными товариществами и сообществами для стимулирования загрузок. Никаких холодных обращений к управляющим компаниям на этом этапе.

---

### Фаза 2 — Построение доверия (месяцы 5–8)

**Цель:** Запустить бенчмаркинг, конвертировать ранних пользователей в платных подписчиков, стать основным инструментом для жилищных товариществ.

**Что выходит:**
- Сравнение между домами (серия × район × площадь)
- Оповещения об аномалиях (переплата, расхождение показаний счётчиков)
- Интеграция Smart-ID для верификации пользователей
- B2B-тариф для товариществ
- Автоматические сводки оповещений для правлений

**Метрики успеха:**
- 50+ платных подписок от товариществ
- 30% пользователей, увидевших оповещение об аномалии, совершают хотя бы одно действие
- База бенчмаркинга охватывает ≥3 серии домов в Риге

---

### Фаза 3 — Рост капитала (месяц 9+)

**Цель:** Запустить полный модуль реновации, закрыть первую сделку с подрядчиком, наладить конвейер документов Altum.

**Что выходит:**
- ИИ-прогноз экономии (счёт за отопление до/после реновации)
- Калькулятор субсидии Altum
- Генерация документов (протоколы собраний, заявки на субсидию)
- Электронное голосование через Smart-ID/eParaksts
- Маркетплейс подрядчиков (первые 10–20 верифицированных компаний)
- Первый закрытый контракт на реновацию

**Метрики успеха:**
- 3+ проекта реновации инициированы
- 1+ контракт на реновацию закрыт
- Среднее время от аномалии аудита до старта реновации: <6 месяцев

---

### Фаза 4 — Масштабирование

**Цель:** Расширить маркетплейс подрядчиков, добавить интеграцию банковского финансирования, выйти на полное национальное покрытие.

**Что выходит:**
- Партнёрство с банком для кредитования реновации (совместное финансирование с Altum)
- Рейтинговая система подрядчиков на основе завершённых проектов
- Мобильное приложение
- B2B API для управляющих компаний
- Расширение за пределы Риги на другие города Латвии

## Риски и меры по снижению

| Риск | Мера |
|------|------|
| Юридическая сила электронного голосования для Altum | Получить юридическое заключение до запуска Фазы 3; валидировать на одном тестовом кейсе |
| Противодействие управляющих компаний | Позиционировать аудит как инструмент жителей; не выходить напрямую против них |
| Низкое качество данных из загружаемых PDF | Настраивать промпт GPT-4o для крайних случаев; добавить fallback с ручной проверкой при `parseConfidence: LOW` |
| Медленное накопление данных в Фазе 1 | Привлечь 2–3 активных товарищества в качестве якорных пользователей |
| Изменения программы Altum | Мониторить объявления Altum; строить конвейер документов на настраиваемых шаблонах |
