## Публичные реестры

### Kadastrs.lv
**Провайдер:** Valsts zemes dienests (Государственная земельная служба)
**Доступ:** Открытые данные (data.gov.lv, CC BY) + контрактный API (требует соглашения с VZD)
**Данные:**
- Адрес и кадастровый номер здания
- Год постройки
- Общая площадь
- Серия здания (для старых домов)
- Класс энергоэффективности (A–G) — через отдельный набор BVKB
- Количество этажей, квартир
**Использование в ALTEKO:** Сегментация зданий для бенчмаркинга; входные данные для ИИ-прогноза
**Примечания:** Открытые данные Building.ZIP (еженедельное обновление) — основной путь без контракта. Контрактный API VZD — для более богатых данных в реальном времени, оценить после MVP.

#### Соотношение VAR и Building.ZIP (верифицировано, май 2026)

В нашей БД после синка обоих источников:
- **573 142** всего зданий
- **365 799** (64%) — реальный кадастровый номер (из Building.ZIP)
- **207 343** (36%) — суррогатный код `VAR:xxx` (нет совпадения в Building.ZIP)

**Почему 36% зданий из VAR отсутствуют в Building.ZIP:**

Адресный реестр VAR и кадастровая система (KIS) — независимые реестры с разным охватом.
Поле `PLAN_ADR` в VAR определяет связанность с KIS [ВЕРИФИЦИРОВАНО]:

> «Признак того, что это плановый адрес. Значение 'Y' — плановый адрес (адрес не привязан ни к одному объекту в Кадастровой информационной системе); значение 'N' — адрес привязан к объекту недвижимости в KIS.»
> — VAR CSV Specification v7.0, раздел 2.2.12, VZD, июль 2025
> https://www.vzd.gov.lv/sites/vzd/files/media_file/varis_csv_specifikacija_v7_24072025_3.pdf

По латвийскому праву (MK Nr. 455, Adresācijas noteikumi, 2021, §5.10 и §12), муниципалитет обязан присвоить адрес строящемуся зданию **до начала согласования строительного замысла** — то есть за месяцы или годы до кадастровой регистрации [ВЕРИФИЦИРОВАНО, likumi.lv/ta/id/324387].

Таким образом, 207k зданий с `VAR:xxx` кодом — это прежде всего:
1. **Плановые адреса** (PLAN_ADR="Y"): адрес присвоен, здание ещё не построено или не зарегистрировано в кадастре
2. **Малые хозяйственные постройки** (сараи, гаражи, ≤60 м²): регистрация через упрощённую процедуру, часто не завершённую
3. **Сельские строения** старой постройки без кадастровой регистрации

Это подтверждается нашими данными: адреса VAR:xxx зданий — преимущественно сельские novads (Alūksnes, Gulbenes, Gulbenes и т.п.), без метаданных (год, площадь, материал = NULL).

**Для ALTEKO критично [ВЕРИФИЦИРОВАНО из собственных данных]:**

Среди 207k VAR:xxx зданий **ноль** многоквартирных домов (≥3 квартир). Все советские многоквартирные дома (128 951 здание с годом постройки 1946–1991) имеют реальный кадастровый номер. Охват кадастра для целевого сегмента ALTEKO — **100%**.

По данным CSDILA Cadastral Template 2015 (Latvia), около 96% всех зданий в Латвии зарегистрированы в KIS [ВЕРИФИЦИРОВАНО, частично устарело — данные 2015 г.; более свежих официальных цифр не найдено]: https://cadastraltemplate.org/latvia.php

### Zemesgrāmata
**Провайдер:** Tiesu administrācija (Судебная администрация)
**Доступ:** Только веб-портал; ~€5 за запрос; пакетного API нет
**Данные:**
- Актуальные собственники квартир (имя, доля %)
- История сделок
- Обременения (ипотеки, ограничения)
**Использование в ALTEKO:** Верификация права на голосование; взвешивание долей собственности в протоколах голосования
**Примечания:** Программного пакетного API не существует. Для Фазы 3 (голосование): ручная загрузка CSV правлением товарищества (предпочтительно) или нотариальная выписка.

---

### Altum.lv
**Провайдер:** ALTUM (государственный институт развитого финансирования)
**Доступ:** Публичный сайт; подтверждённого структурированного API нет
**Данные:**
- Действующие субсидийные программы и условия
- Допустимые типы зданий
- Требования к заявке и контрольные списки документов
- Список одобренных проектов (публичный)
**Использование в ALTEKO:** Расчёт права на субсидию; генерация конвейера документов
**Примечания:** Данные, вероятно, потребуют ручного обновления или скрейпинга. Открытого API не подтверждено.

---

### CSP.gov.lv
**Провайдер:** Centrālā statistikas pārvalde
**Доступ:** Публичные наборы данных (CSV/XLSX для скачивания)
**Данные:**
- Индексы строительных затрат (по году, региону)
- Статистика жилищного фонда
- Демографические данные
**Использование в ALTEKO:** Скорректированные на инфляцию оценки стоимости ремонта; оценка объёма рынка
**Примечания:** Подходит для пакетного скачивания, не для интеграции в реальном времени.

### GeoRiga / GeoLatvija
**Провайдер:** Рижская городская дума / VSIA «Latvijas Ģeotelpiskās informācijas aģentūra»
**Доступ:** opendata.riga.lv / geolatvija.lv (частично открытый)
**Данные:**
- Контуры зданий и геопространственные координаты
- Границы районов
- Реестр адресов
**Использование в ALTEKO:** Сегментация по районам; визуализация на карте
**Примечания:** Статус доступа к конкретным наборам данных требует проверки.

## Платные провайдеры данных

### Lursoft
**Провайдер:** Lursoft IT (коммерческий)
**Доступ:** Платная подписка / API
**Данные:**
- Данные о регистрации компании
- Годовая финансовая отчётность
- Структура собственности юридических лиц
- Показатели платёжного поведения
**Использование в ALTEKO:** Верификация подрядчиков — проверка финансового состояния перед включением в маркетплейс
**Примечания:** Платный API. Стоимость и условия требуют коммерческого соглашения.

### Jāņa sēta API
**Провайдер:** Jāņa sēta (коммерческий, kartes.lv)
**Доступ:** API-ключ (бесплатный тариф доступен, email: kartes@kartes.lv)
**Данные:**
- Автодополнение адреса (поиск по строке)
- Геокодинг (адрес → lat/lon + vzd_id)
- Обратное геокодирование с кадастровым поиском
- API квартир (номера квартир по vzd_id здания)
**Использование в ALTEKO:** Основной провайдер для автодополнения адреса и геокодинга в модуле поиска по адресу
**Примечания:** Оборачивает официальные данные VAR (реестр адресов VZD) через чистый REST API.

## Открытые данные о сделках и кадастровых стоимостях

### NITIS (Nekustamā īpašuma tirgus datu bāze)
**Провайдер:** Valsts zemes dienests (VZD)
**Доступ:** Открытые данные, CC-BY-4.0, бесплатное скачивание
**URL:** data.gov.lv/dati/lv/dataset/nekustama-ipasuma-tirgus-datu-bazes-atvertie-dati
**Формат:** CSV + XLSX (UTF-8 BOM), обновление 10-го числа каждого месяца
**Данные:**
- Все сделки с недвижимостью с 2012 г. по настоящее время, более 1 млн записей
- Поля: адрес, кадастровый номер, цена, площадь м², дата, тип сделки, год постройки, износ (добавлен февраль 2025)
**Использование в ALTEKO:** Медианный €/м² по району/серии/году — основа ценового бенчмаркинга в модуле аудита [НЕ ВЕРИФИЦИРОВАНО: фактическое покрытие данных и полнота полей требуют проверки до интеграции]
**Примечания:** Первоначальная загрузка архивов 2012–2026, затем ежемесячный инкрементальный синк.

---

### Darījumi ar telpu grupām (Сделки с квартирами)
**Провайдер:** VZD
**Доступ:** Открытые данные, CC-BY-4.0
**URL:** data.gov.lv/dati/lv/dataset/darijumi-ar-telpu-grupam
**Формат:** CSV + XLSX, обновление 10-го числа каждого месяца
**Данные:**
- Сделки с квартирами с 2016 г. по настоящее время
- Поля: адрес квартиры, цена, площадь м², этаж, количество комнат, дата, координаты здания
**Использование в ALTEKO:** Более детализированные ценовые данные на уровне квартирных характеристик; дополняет NITIS для модуля аудита [НЕ ВЕРИФИЦИРОВАНО: охват по городам и полнота данных требуют проверки]
**Примечания:** Перекрывается с NITIS, но содержит более детальные характеристики квартиры. Оценить возможность совместной загрузки.

### PremiseGroup.ZIP (Telpu grupu raksturojošie dati)
**Провайдер:** VZD, часть открытых данных Кадастровой информационной системы
**Доступ:** Открытые данные, CC-BY-4.0, тот же ZIP-архив, что и Building.ZIP
**URL:** data.gov.lv/dati/lv/dataset/kadastra-informacijas-sistemas-atvertie-dati
**Формат:** XML + XSD-схема, еженедельное обновление
**Данные:**
- Характеристики квартир и групп помещений: площадь м², этаж, количество комнат, код адреса, кадастровый номер
**Использование в ALTEKO:** Автоматическое определение площади квартиры без ввода пользователем; повышает точность бенчмарка [НЕ ВЕРИФИЦИРОВАНО: корректность площадей и актуальность данных требуют проверки]
**Примечания:** Загружается вместе с Building.ZIP в рамках того же pipeline. Проверить совместимость кадастровых номеров с NITIS для JOIN-операций.

### Кадастровые стоимости 2025 (Fiskālās un universālās kadastrālās vērtības)
**Провайдер:** VZD
**Доступ:** Открытые данные, CC-BY-4.0
**URL:** data.gov.lv/dati/dataset/fiskalas-un-universalas-kadastralas-vertibas-uz-2025-gada-1-janvari
**Формат:** CSV + JSON, ежегодная публикация (1 января)
**Данные:**
- Фискальная стоимость (база 2012–2013 г., для налогообложения) и универсальная стоимость (база июль 2022 г., не более 80% рыночной) на кадастровую единицу
**Использование в ALTEKO:** Сравнение «официальная стоимость vs. рыночная цена» в модуле аудита [НЕ ВЕРИФИЦИРОВАНО: методология расчёта и актуальность для советских домов требуют уточнения]
**Примечания:** Обновляется раз в год. Необходимо хранить исторические версии для трендового анализа.

### Зоны кадастровых стоимостей (Kadastrālo vērtību zonējumi)
**Провайдер:** VZD
**Доступ:** Открытые данные, CC-BY
**URL:** data.gov.lv/dati/eng/dataset/speka-esosie-kadastralo-vertibu-zonejumi
**Формат:** SHP (Shapefile), нерегулярные обновления
**Данные:**
- Пространственные границы зон кадастровых стоимостей с базовыми ставками за м²
**Использование в ALTEKO:** Картографическая визуализация — окраска зон по кадастровой стоимости [НЕ ВЕРИФИЦИРОВАНО: требует PostGIS или конвертации в GeoJSON; стоимость реализации требует оценки]
**Примечания:** Требует PostGIS или предварительной конвертации в GeoJSON для использования в Next.js. Рассмотреть на более поздних фазах.

### Индексы цен CSP (House Price Index & Construction Cost Index)
**Провайдер:** Centrālā statistikas pārvalde (CSP/CSB)
**Доступ:** Бесплатно, доступен API
**URL:** stat.gov.lv/en/api-and-code-list-registry/api
**Формат:** API + скачивание CSV
**Данные:**
- Индекс цен на жильё по кварталам и регионам
- Индекс строительных затрат по месяцам
**Использование в ALTEKO:** Оценки стоимости реновации с поправкой на инфляцию; контекст рыночных тенденций [НЕ ВЕРИФИЦИРОВАНО: детализация по типам зданий и историческая глубина требуют проверки]
**Примечания:** Дополняет уже задокументированный раздел CSP.gov.lv (публичные наборы данных). Преимущество — наличие API для автоматической актуализации.

## Пользовательские данные

### PDF с расходами (загружаются пользователями)
**Источник:** Ежемесячные/квартальные отчёты управляющих компаний
**Формат:** PDF (сканированный или цифровой)
**Обработка:** OCR → LLM-парсинг → структурированная таблица расходов
**Извлекаемые данные:**
- Отопление (siltumenerģija) — €/мес., €/м²
- Холодная вода, горячая вода — м³, €
- Водоотведение
- Уборка / обслуживание территории
- Взносы в ремонтный фонд здания
- Административные сборы
**Примечания:** Основной канал сбора данных для формирования базы бенчмаркинга. Качество сильно варьируется между управляющими компаниями.

## Погода и климат

### Weather API
**Рекомендуемый провайдер:** Open-Meteo (бесплатно, open-source)
**Необходимые данные:**
- Исторические градусо-дни отопления (HDD) по местоположению
- Среднемесячные температуры по районам
**Использование в ALTEKO:** ИИ-прогноз тепловой экономии (моделирование теплопотребления)
**Примечания:** Open-Meteo предоставляет исторические данные бесплатно с достаточным покрытием для Латвии.
